# Standard Operating Procedure (SOP): BirdNET Acoustic Detection Pipeline

**Version:** 1.0  
**Date:** May 22, 2026  
**Status:** Operational  

---

## 1. Purpose and Scope
This pipeline provides an automated, end-to-end workflow for acquiring avian acoustic recordings, processing them via the BirdNET-Analyzer machine learning model, and validating the results against ground-truth metadata. 

**Scope:**
- **Data Source:** xeno-canto API (v3).
- **Geographic Focus:** Connecticut / Northeastern United States.
- **Target Species:** 10 common Connecticut species (e.g., American Robin, Black-capped Chickadee).
- **Target Audience:** Biological technicians and data scientists performing acoustic monitoring and inventory.

## 2. Prerequisites

### Software Dependencies
The pipeline requires the following environment:
- **Python 3.11+**
- **R 4.x** (for downstream analysis and reporting)
- **ffmpeg:** Must be installed and available in the system PATH for audio conversion.
- **Python Packages:** `birdnetlib`, `pandas`, `requests`, `python-dotenv`, `pydub`.
- **R Packages:** `ggplot2`, `dplyr`, `readr`, `rmarkdown`.

### Environment Configuration
A `.env` file must exist in the root directory containing your xeno-canto API key:
```env
XC_API_KEY=your_api_key_here
```

### Directory Structure
The following structure is required and will be partially generated by the scripts:
- `data/raw/`: Original MP3 files.
- `data/wav/`: Converted WAV files.
- `data/processed/`: Per-file detection results.
- `metadata/`: Master ingestion log.
- `outputs/`: Validated results and metrics.
- `scripts/`: Pipeline execution scripts.

## 3. Data Acquisition (`01_acquire.py`)
This script queries the xeno-canto API for high-quality (Quality A) recordings of the target species.

### Execution
Run the script from the root directory:
```bash
python scripts/01_acquire.py
```

### Modification for Different Regions
To monitor a different region:
1. Open `scripts/01_acquire.py`.
2. Update the `SPECIES` list with the desired common names, scientific names, and 8-character codes.
3. Update the `query` variable in `acquire_data()` if you need to change the country (`cnt:"United States"`) or quality filter.

### Expected Outputs
- `data/raw/*.mp3`: Downloaded audio files.
- `metadata/ingestion_log.csv`: Initialized with metadata for each downloaded file.

### Troubleshooting
- **0 results:** Verify the scientific name is spelled correctly and that the species exists in the xeno-canto database for the specified country.
- **Download failures:** Ensure `XC_API_KEY` is valid and you have an active internet connection.

---

## 4. Audio Conversion (`02_convert.py`)
BirdNET-Analyzer requires audio in a specific format for optimal detection performance. This script uses `ffmpeg` to standardize the recordings.

### Technical Specifications
- **Format:** WAV
- **Sample Rate:** 48,000 Hz (48kHz)
- **Channels:** Mono
- **Bit Depth:** 16-bit PCM

### Execution
```bash
python scripts/02_convert.py
```

### Verification
- Check `data/wav/` for the corresponding `.wav` files.
- The `filename_wav` column in `metadata/ingestion_log.csv` will be updated with the filename or "FAILED".

---

## 5. BirdNET Processing (`03_process.py`)
This script runs the `birdnetlib` wrapper around the BirdNET-Analyzer model to identify bird vocalizations.

### Model Logic
- **Segmenting:** Audio is analyzed in 3-second segments.
- **Filtering:** The model uses provided latitude, longitude, and date to filter its species predictions based on known occurrence ranges.
- **Parameters:**
  - Latitude: 41.6032 (CT)
  - Longitude: -73.0877 (CT)
  - Date: June 15 (Representative summer/breeding season date)
  - Min Confidence: 0.1 (Raw threshold for later filtering)

### Execution
```bash
python scripts/03_process.py
```

### Runtime and Output
- **Runtime:** Expect ~5-10 seconds per minute of audio, depending on CPU/GPU.
- **Output:** A CSV file for every processed WAV in `data/processed/`, containing `start_time`, `end_time`, `common_name`, and `confidence`.

---

## 6. QA/QC Validation (`04_qaqc.py`)
Validation compares BirdNET's "blind" detections against the "known" identity of the audio from xeno-canto.

### Confidence Threshold Framework
The pipeline evaluates detections at four thresholds: 0.25, 0.50, 0.75, and 0.90.
- **Precision:** Higher thresholds reduce false positives but increase "missed" detections.
- **Recall:** Lower thresholds capture more real birds but introduce more "noise" (false positives).

### Operational Standard
- **Recommended Threshold:** **0.50**
- **Rationale:** A 0.50 threshold provides a balanced tradeoff for Connecticut species, maintaining high recall for common vocalizations while keeping the False Positive Rate (FPR) manageable for occupancy modeling and presence/absence analysis.

### Execution
```bash
python scripts/04_qaqc.py
```

### Outputs
- `outputs/threshold_metrics.csv`: Detailed performance stats across thresholds.
- `outputs/detections_validated.csv`: The final "clean" dataset for ecological analysis.

## 8. Replication Checklist
1. **Setup Environment:** Install Python 3.11, R 4.x, and ffmpeg.
2. **Install Dependencies:** `pip install -r requirements.txt`.
3. **Configure API:** Create `.env` with `XC_API_KEY`.
4. **Acquire Data:** Run `python scripts/01_acquire.py`.
5. **Convert Audio:** Run `python scripts/02_convert.py`.
6. **Process Detections:** Run `python scripts/03_process.py`.
7. **Validate Results:** Run `python scripts/04_qaqc.py`.
8. **Summarize:** Run `python scripts/05_combine.py`.
9. **Analyze:** Execute R scripts in `r_analysis/` for final reports.
